reboot-to-dev

This repository contains three files: reboot-to-dev-mode, reboot-to-user-mode, and issue.dev.

Usage

The scripts reboot-to-dev-mode and reboot-to-user-mode are used to switch the system between development mode and user mode. They should be run as root, and will prompt the user for confirmation before making any changes.

issue.dev is a warning message displayed to the user when the system is in development mode. It is copied to /etc/issue when the system is switched to development mode, and is removed when the system is switched back to user mode.

Installation

To install these scripts, run make install. By default, they will be installed in /usr/local/bin and /etc, but you can customize the installation directory by setting the PREFIX and DESTDIR variables. For example, to install the scripts in /opt/devmode/bin, run:

make install PREFIX=/opt/devmode

To uninstall the scripts, run make uninstall.
